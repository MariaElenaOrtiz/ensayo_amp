Introducción

El propósito del ensayo consiste en conocer la funcionalidad de Media Query en el desarrollo de páginas web, el compartamiento en los diferentes tipos de navegadores. Asi mismo daremos un vistazo a la semantica web, el uso de los colores en sus diferentes código (hexadecial, rgb y hsl) y algunas unidades de medida comos son EM y REM.

Otro elemento importante es conocer AMP su estructura y su funcionamiento para sitios webs.

¿Qué es Media Query?

Media Query es un modulos de las hojas de estilo(CSS) utilizada en las páginas web para darles las características deseadas como son el color de fondo, imagen de fondo, tipo de fuente para los encabezados, párafos entre otros. Pero en especial los media querys proporciona el tamaño ideal para que no pierda su estructura y su estilo los páginas web en su consulta utilizando diferentes dispositivos, es decir, podemos consultar una misma página web en una computadora de escritorio "PC", en una computadora portatil "LAPTOP" o en dispositivos inteligentes como pueden ser una "TABLET" o un "SMARTPHONE". Cada uno de estos dispositivos tiene un tamaño de pantalla y resolución, lo cual si no utilizsmos los media query no se adaptaria el contenido a cada uno de ellos y no se visualizaria de forma correcta.

Este es une ejemplo básico de la sintaxis de media query.

@media not|only mediatype and (media feature and|or|not mediafeature) {
CSS-Code;CSS-Code;
}

Fucnionalidad de Media Query

La funcionalidad de los media querys van relacionados con expresiones tales como less than or equal to, greater or equal to, min o max.

Veamos su funcionalidad con relación a los archivos html y CSS.

  • Utilizando la etiqueta "link".

    <link rel="stylesheet" href="estilos-generales.css">

  • Utilizando la etiqueta "link" agregando el atributo media.

    <link rel="stylesheet" href="estilo-imprimir.css" media="print">

    Print. Este atributo aplica sólo cuando se va a imprimir la página.

  • Utilizando media query en la hoja de estilos.

    @media (min-width: 500px) {
    h1{
    margin: 1%;
    }
    .estiloresponsive{
    float: right;
    padding-left: 15px;
    }
    }

  • Utilizando los operadores lógicos AND, OR, NOT y ONLY.

    @media (max-width: 600px) and (orientation: landscape) {
    h1{
    color: red;
    }
    }

    1. AND. Las dos condiciones deben de cumplirse.
    2. OR. Se cumple cualquiera de las condiciones.
    3. NOT. Cuando no se cumple la condicion se aplica el meida query.
    4. ONLY. Sólo aplica en caso que se cumplan ciertas condiciones.

Semantica web

"La semántica web es un conjunto de actividades desarrolladas en el seno de World Wide Web Consortium con tendencia a la creación de tecnologías para publicar datos legibles por aplicaciones informáticas.​ Se basa en la idea de añadir metadatos semánticos y ontológicos a la World Wide Web. En el transcurso del tiempo la semantica ha evolucionado hasta llegar a la web 3.0 que son los metalenguajes y los estándares de representación."

La búsqueda que realizan los usuario se desarrolla no centrándose únicamente en los parámetros buscados, sino en la semántica o significado de los mismos.

Ventajas

  • Incorpora contenido semántico a las páginas que se suben a Internet. Esto permite una mejor organización de la información, asegurando búsquedas más precisas por significado y no por contenido textual.
  • Permite a las computadoras la gestión de conocimiento, hasta el momento reservada a las personas (hace uso de inteligencia artificial).

Desventajas

  • Es necesario unificar los estándares semánticos y proveer relaciones de equivalencia entre conceptos.
  • Es costoso y laborioso adaptar los documentos de Internet, para poder ser procesados de forma semántica (a esto hay que sumar los problemas del idioma).

Colores web

Los colores en HTML son una parte fundamental para su desarrollo ya que con ellos podemos cambiar de tonalidades los textos, enlace y fondos. Contamos con tres tipos de códigos para los colores:

  1. HEX. Utiliza tres bytes números hexadecimal (seis dígitos) cada byte representa la intensidad de rojo, verde y azul en el color.
  2. RGB. Utiliza la combinación de tres colores primarios que son el rojo, verde y azul, utilizan una escala del 0 al 255, ejemplificado de esta forma (0, 0, 0).
  3. HSL. Utiliza la hsl() prefijo y tres valores de tono (escala de 0 a 360), saturación (porcentaje entre el 0% y el 100%) y luminosidad (porcentaje entre el 0% y el 100%).

REM y EM

Las unidades de media EM y REM son traducidos por los navegadores en pixeles. Esta establecido que 16 pixeles son iguales a 1 em o 1 rem (16px = 1em o 16px = 1rem). Em o REM son unidades flexibles y escalables para los diseños de las páginas webs permitiendo que sean responsivas para diferentes dispositivos de consulta.

  • REM

    Esta unidad depende directamente de la raiz de la página, es decir, permite escalar con solo cambiar el elemento raiz html.

    html {
    font-size: 16px;
    }
    h1{
    font-size: 2rem; /* 2 × 16px = 32px */
    margin-top: 1.25rem; /* 1.25rem = 20px, ya que 1.25 × 16 = 20 */
    margin-bottom: 1.25rem;
    }

  • EM

    Esta unidad depende del último font-size definido dentro del árbol de elementos en el que se encuentra.

    CSS

    html {
    font-size: 100%; /* 100% = 16px */
    }
    .container{
    font-size: 1.25em;
    }
    .small-font{
    font-size: 0.9em;
    }

    HTML

    <div class="container">
    <p>Titulo</p>
    <p class="small-font">Texto dentro del contenedor</p>
    </div>
    <hr>
    <p class="small-font">Texto fuera del contenedor </p>

AMP

AMP es una biblioteca de código abierto que ofrece una forma directa de crear páginas web atractivas, ligeras y que se cargan prácticamente al instante para los usuarios.

Las páginas AMP se crean con tres componentes básicos:

  • AMP HTML

    AMP HTML es, básicamente, código HTML extendido con propiedades AMP personalizadas.


    <!doctype html>
    <html ⚡>
    <head>
    <meta charset="utf-8">
    <link rel="canonical" href="hello-world.html">
    <meta name="viewport" content="width=device-width,minimum-scale=1,initial-scale=1">
    <style amp-boilerplate>body{-webkit-animation:-amp-start 8s steps(1,end) 0s 1 normal both;-moz-animation:-amp-start 8s steps(1,end) 0s 1 normal both;-ms-animation:-amp-start 8s steps(1,end) 0s 1 normal both;animation:-amp-start 8s steps(1,end) 0s 1 normal both}@-webkit-keyframes -amp-start{from{visibility:hidden}to{visibility:visible}}@-moz-keyframes -amp-start{from{visibility:hidden}to{visibility:visible}}@-ms-keyframes -amp-start{from{visibility:hidden}to{visibility:visible}}@-o-keyframes -amp-start{from{visibility:hidden}to{visibility:visible}}@keyframes -amp-start{from{visibility:hidden}to{visibility:visible}}</style><noscript><style amp-boilerplate>body{-webkit-animation:none;-moz-animation:none;-ms-animation:none;animation:none}</style></noscript>
    <script async src="https://cdn.ampproject.org/v0.js"></script>
    </head>
    <body>Hello World!</body>
    </html>

  • JavaScript de AMP

    La biblioteca JavaScript de AMP gestiona la carga de recursos todo para que las páginas se rendericen rápidamente.

  • Caché de AMP

    La caché de AMP de Google es una red de distribución de contenido basada en proxies con la que se entregan documentos AMP válidos. Cuando se utiliza la caché de AMP de Google, se cargan el documento, todos los archivos JavaScript y todas las imágenes del mismo origen que esté utilizando HTTP 2.0 para maximizar la eficiencia.
